# system-design-journal

Notes from a 6-month journey through system design — starting 28th June 2026

## Structure

- **`Phase-1/`** — daily notes, in depth, one concept (or small group of concepts) per day.
- **`Reference/`** — standalone cheat sheets for fast recall under time pressure. Start with [`Reference/top-30-system-design-concepts.md`](Reference/top-30-system-design-concepts.md) — the 30 foundational concepts every system design problem draws from, each with a diagram, trade-offs, and a one-line interview answer.
